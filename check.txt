====================================================================
I. KIẾN TRÚC ROLE THỰC TẾ
====================================================================

Hệ thống gồm 3 role chính:

1. Guest (khách chưa đăng nhập)
- Xem danh sách phòng.
- Tìm kiếm phòng.
- Lọc phòng.
- Xem chi tiết phòng.
- Xem thông tin liên hệ.
- Đăng ký tài khoản.
- Đăng nhập.
- Gửi liên hệ/report nếu cho phép.

2. User (người thuê phòng)
- Đăng ký tài khoản thuê.
- Đăng nhập/logout.
- Quản lý hồ sơ cá nhân.
- Lưu phòng yêu thích.
- Đặt lịch xem phòng.
- Booking phòng nếu hệ thống có.
- Thanh toán nếu có.
- Viết đánh giá.
- Báo cáo phòng vi phạm.
- Đổi mật khẩu.
- Quên mật khẩu.

3. Owner (chủ trọ/chủ phòng)
- Đăng ký tài khoản owner.
- Chờ admin duyệt.
- Đăng nhập/logout.
- Quản lý hồ sơ owner.
- Tạo phòng trọ.
- Sửa phòng trọ.
- Upload ảnh.
- Quản lý booking.
- Theo dõi doanh thu.
- Theo dõi đánh giá.
- Quản lý trạng thái phòng.
- Gia hạn tin nếu có.
- Xem lý do phòng bị reject/ẩn.

4. Admin
- KHÔNG phải người đăng phòng.
- KHÔNG sửa nội dung phòng của owner.
- Chỉ làm nhiệm vụ vận hành hệ thống:
  - quản lý user
  - duyệt owner
  - kiểm duyệt phòng
  - xử lý report
  - xử lý booking/payment
  - quản lý danh mục
  - xem dashboard
  - theo dõi log
  - khóa tài khoản
  - xử lý vi phạm

II. HỆ THỐNG LOGIN/LOGOUT CHUẨN THỰC TẾ
====================================================================

Login dùng chung:
- public/login.php

Redirect:
- admin -> public/admin/index.php
- owner -> public/owner/dashboard.php
- user -> public/user/dashboard.php

Session bắt buộc:
- $_SESSION['user_id']
- $_SESSION['role']
- $_SESSION['user_role']
- $_SESSION['user_name']

Yêu cầu:
1. Login bằng email + password.
2. Password dùng:
   - password_hash
   - password_verify
3. Không so sánh plain text.
4. session_regenerate_id(true) sau login thành công.
5. Logout phải destroy session hoàn toàn.
6. Không cho back lại dashboard sau logout.
7. Middleware/admin_init bắt buộc check:
   - login
   - role
   - blocked
8. Toàn bộ route admin phải bảo vệ server-side.
9. Không chỉ ẩn nút bằng frontend.

III. CAPTCHA & LOGIN SECURITY CHUẨN WEBSITE THỰC TẾ
====================================================================

Flow bảo mật giống website thật:

1. Ban đầu KHÔNG hiển thị captcha.
2. Sai login lần 1:
   - báo “Email hoặc mật khẩu không đúng.”
3. Sai lần 2:
   - vẫn báo lỗi chung.
4. Sai lần 3:
   - bật trạng thái yêu cầu captcha.
5. Từ lần 4:
   - bắt buộc captcha.
6. Captcha validate SERVER-SIDE.
7. Sai captcha:
   - tạo captcha mới.
8. Sai quá nhiều:
   - khóa tạm:
     - 1 phút
     - 5 phút
     - 15 phút
9. Login thành công:
   - reset failed_count
   - reset captcha_required
   - reset lock
10. Ghi log:
   - login success
   - login failed
   - IP
   - user agent
   - thời gian

Captcha:
- Ưu tiên:
  - Cloudflare Turnstile
  - Google reCAPTCHA
- Có fallback local captcha để demo local/XAMPP.

Bắt buộc:
- CSRF token
- validate input
- prepared statement
- escape HTML output

V. QUÊN MẬT KHẨU / RESET PASSWORD
====================================================================

1. Quên mật khẩu bằng token.
2. Token có expiry.
3. Token chỉ dùng 1 lần.
4. Không tiết lộ email tồn tại hay không.
5. Admin có thể reset password user/owner.
6. Sau reset:
   - force change password khi login lần đầu.
