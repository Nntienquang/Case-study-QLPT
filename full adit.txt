3. DATABASE AUDIT
============================================================

Bang co cau truc kha on:
- users: co PK, unique email, unique phone, role/status index. Nhung status can chuan hoa.
- motels: co PK, index owner/category/district/status/price-area/location. Nhung data denormalized nhieu.
- bookings: co booking_code unique, index user/owner/motel/status/payment_status. Nhung duplicate status nghiem trong.
- payments: co payment_code unique, index booking/status/transaction_code. Nhung duplicate payment status.
- favorites: co unique user_id/motel_id.
- wallets: unique user_id.
- login_security_state/login_attempts: dung cho brute force, co index hop ly.
- page_views: schema phu co index theo viewed_at/page/user.
- owner_warnings: schema phu co index owner/admin/level.

Bang can sua/bo sung:
- bookings:
  - Co ca status va booking_status, nhieu luc mapping khong dong nhat: accepted vs confirmed, paid vs waiting_payment.
  - deposit_amount/total_amount/final_amount/voucher_id duoc ADD COLUMN sau dump, can migrate versioned.
  - cancelled_by/rejected_by khong co FK.
  - check_in_date, expected_move_in_date, checkin_date trung y nghia.
- payments:
  - Co ca payment_status va status. payment_status la payment lifecycle; status la escrow lifecycle. Nen doi ten status thanh escrow_status.
  - payment_method va method trung y nghia.
- users:
  - status varchar cho approved/pending/active/blocked/rejected nhung register user tao 'active', login/admin dung 'approved'. Can chuan hoa.
  - owner_verification_status tach rieng la dung, nhung public/owner/profile.php co the tu set approved.
  - reset_token plain text trong users, trong khi co password_resets table chua duoc dung dung muc.
  - CCCD/bank/KYC file nam cung users, nen tach owner_profiles/owner_kyc_documents neu production.
- motels:
  - address + province/district/ward/street denormalized. Chap nhan de search, nhung can constraint va sync rule.
  - utilities text va motel_utilities song song, de lech du lieu.
  - amenities_json them sau dump nhung app van dung utilities/motel_utilities.
  - money int ok voi VND nho, nhung production nen BIGINT UNSIGNED/DECIMAL cho tien.
- favorites vs wishlists:
  - Duplicate concept. Public detail check favorites, AJAX/saved-motels/search dung wishlists. Gay UX favorite bi sai.
- monthly_bills vs monthly_invoices:
  - Hai bang hoa don thang co overlap. App chu yeu dung monthly_bills, monthly_invoices gan nhu chua dung.
- reports/reviews/viewing_appointments/maintenance_requests:
  - status dang varchar/khong enum dong nhat. Nen chuan hoa bang enum/app constants.
- transactions:
  - from_user/to_user nullable nhung khong co ledger invariant. Can transaction_type va signed amount ro hon.
- vouchers/wishlists duoc tao sau AUTO_INCREMENT va truoc COMMIT trong dump, khong co FK/index phu motel_id.

Van de FK/index:
- Gan nhu khong co FOREIGN KEY thuc thi. Rui ro orphan rows rat cao khi xoa motel/user/payment.
- Motel::delete() xoa cascade bang code, nhung khong xoa payments, transactions, conversations, messages, booking_room_holds, monthly_bills, reports day du.
- Can index bo sung:
  - motels(status, room_status, district_id, category_id, price, area)
  - bookings(owner_id, status, created_at), bookings(user_id, created_at)
  - payments(payment_status, status, created_at), payments(booking_id, payment_status)
  - reports(status, created_at), reviews(motel_id, status, created_at)
  - viewing_appointments(owner_id, status, preferred_time)
  - wishlists(motel_id)

SQL/migration de xuat uu tien:

-- Doi ten escrow status neu duoc phep migrate breaking change

7. SECURITY AUDIT
============================================================

Critical:
1. Owner KYC bypass:
   - public/owner/profile.php tu set owner_verification_status='approved' khi co idcard/anh/bank. Owner co the tu duyet minh, bo qua admin.
2. File upload risk:
   - public/owner/add-listing.php, edit-listing.php, owner/profile.php, user/profile.php dung extension/pathinfo va move_uploaded_file; thieu MIME sniffing, size limit, image re-encode, random extension whitelist day du, virus scan, storage ngoai webroot.
3. CSRF missing:
   - register.php, owner-register.php, forgot.php, reset.php, checkout.php, motel-detail.php POST, owner revenue/profile/settings/listing forms, AJAX toggle-favorite/submit-review.
4. DB dump contains credentials/demo accounts and one plain password hash-invalid row.
5. No global security headers: CSP, X-Frame-Options/frame-ancestors, nosniff, Referrer-Policy, HSTS for HTTPS.

High:
- IDOR mostly guarded by owner_id/user_id in major pages, but can audit tat ca endpoints.
- Raw SQL concatenation still exists. Many are cast int, but owner/bookings notification insert concat title/body unsafe.
- Webhook permits trusted IP using X-Forwarded-For without trusted proxy validation. If app directly exposed, header spoof risk.
- Password reset token plain text, no token hash, no per-email/IP rate limit.
- No role middleware central for user pages.
- No upload path traversal from filename due uniqid, but extension can include odd values if not lower/whitelisted.

Medium:
- Captcha custom fallback only sau failures; register captcha basic.
- Remember email cookie lacks Secure/SameSite flags.
- Session cookie config not hardened.
- Notifications redirect_link in public/notifications.php redirects raw POST link, potential open redirect/internal path issue.
- Activity logs store user_agent/ip, fine, but no retention/privacy policy.
